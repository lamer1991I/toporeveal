0. CONTEXTO DEL PROYECTO
TopoReveal es un visualizador de topologia de red en tiempo real. Detecta, fingerprinta y visualiza todos los dispositivos de una red local combinando captura pasiva (scapy) y escaneo activo (nmap). UI con tema oscuro GitHub, construido en Python + tkinter. Probado en red universitaria con 181 hosts activos.
Ruta del proyecto: ~/Proyectos/toporeveal/
Estructura de directorios
~/Proyectos/toporeveal/
├── assets/icons/          (21 PNG neon — router/pc/smartphone/switch/etc)
├── core/
│   ├── nodes.py           (Nodo dataclass + timers + risk score + hallazgos)

1. ESTADO ACTUAL — QUE ESTA TERMINADO Y QUE NO
Completado al 100%
    • Scanner multi-hilo con escala adaptativa: 2 workers (<20 hosts), 3 workers (20-80), 5 workers (>80 hosts). El monitor de escala corre cada 60s y lanza workers adicionales si la red crece.
    • Intervalos adaptativos: ARP cada 5-8 min segun escala, re-scan profundo cada 8-20 min.
    • 27 puertos escaneados: criticos (23/Telnet, 5900/VNC, 445/SMB, 3389/RDP), servicios (22/80/443/8080), IoT/OT (502/Modbus, 102/S7, 1883/MQTT), BD (3306/5432/1433/27017).
    • Captura pasiva scapy con watchdog thread (auto-restart hasta 10 veces si cae).
    • Topologia visual en arbol: Internet → Gateway → Switches → Hosts.
    • Estados con timers balanceados: 2 min→sospechoso, 5 min→fantasma, 10 min→lobby.
    • FIX lobby timer: trafico dirigido HACIA un nodo (solo_destino=True) NO resetea su timer de inactividad. Solo el trafico FROM el nodo resetea el timer.
    • FIX NoneType: stats_hash en panel_alertas.py usa str(n.tipo or "") para evitar comparar None con str en sorted().
    • 21 iconos PNG neon por tipo de dispositivo en assets/icons/.
    • Sistema hallazgos con risk score 0-100 + acciones sugeridas por servicio.
    • Panel alertas izquierdo con semaforo por max(severidad), no promedio.
    • Filtro por tipo: click en tipo en panel izquierdo → esos nodos se iluminan rojo en canvas, el resto se atenua.
    • Arsenal: click derecho en cualquier nodo → menu contextual → 8 tipos de escaneo → ventana flotante con output nmap coloreado.
Parcialmente implementado
    • Fingerprinting OS (82%): TTL funciona bien para Linux/Windows/iOS. MAC OUI tiene cobertura parcial. Falta ampliar la base de prefijos OUI.
    • Export (85%): PNG y JSON funcionan. CSV exporta pero falta llamar a makedirs() antes de crear el archivo. Bug activo que hace fallar el export si la carpeta exports/ no existe.
    • Scanner logging (90%): log() imprime en consola. set_log_callback() existe pero a veces el callback se ignora en ramas de error del scanner.
No implementado (0%)
    • SQLite historial diferencial entre sesiones
    • Dashboard de estadisticas con graficas matplotlib
    • GeoIP para conexiones externas
    • Multi-subnet / VLAN detection
    • Informe PDF profesional
    • IPv6

Clase Nodo — campos criticos
CAMPO	TIPO	DESCRIPCION
ip	str	IP del nodo (clave primaria)
mac	str|None	MAC address (puede llegar tarde via ARP)
tipo	str	router/pc/smartphone/switch/servidor/arp-scanner/...
estado	str	confirmado / sospechoso / fantasma
en_lobby	bool	True = invisible en canvas (>10 min inactivo)
ultimo_visto	float	time.time() — SOLO actualiza si es ORIGEN del trafico
puertos_abiertos	list[int]	Puertos TCP detectados por nmap
hallazgos	list[Hallazgo]	Vulnerabilidades/servicios detectados
risk_score	int 0-100	Calculado por calcular_risk_score()
severidad_max	str|None	info/medio/alto/critico o None si sin hallazgos
protegido	bool	True = IP local, nunca va a lobby
perfiles	list	Roles: [(nombre, severidad)] — ej: ("Posible DC", "critico")

6. REGLAS DE ARQUITECTURA — NUNCA ROMPER
    • NUNCA actualizar la UI desde un hilo secundario. Usar ventana.after(0, lambda: fn()) desde todos los callbacks de scanner y capture.
    • nodo.actualizar_actividad() SOLO cuando el nodo es ORIGEN del trafico. Para destino usar registrar_paquete_destino(). Si no se respeta esto, todos los nodos se mantienen "activos" porque el gateway hace ARP a todos constantemente.
    • El hash-based rendering en panel_alertas.py es INTENCIONAL. No eliminarlo. Sin el, en redes de 180+ hosts el panel parpadea cada 2 segundos y se vuelve inutilizable.
    • El stats_hash DEBE usar str() sobre tipo y severidad_max para evitar el NoneType bug que aparece cuando hay nodos recien descubiertos sin hallazgos aun.
    • Los iconos PNG deben estar en assets/icons/{tipo}.png. Si falta el PNG el canvas tiene fallback con emoji de texto — no crashea.
    • El Arsenal usa multi-listener: cada VentanaArsenal se registra con arsenal.anadir_listener() y se desregistra al cerrarse. No reemplazar por un callback unico global.
Patron correcto de comunicacion UI <-> Backend
# CORRECTO — desde hilo secundario:
def _callback_scanner(self, datos):
    self.ventana.after(0, lambda: self._procesar_en_ui(datos))

# INCORRECTO — crashea o bloquea tkinter:
def _callback_scanner(self, datos):
    self._procesar_en_ui(datos)  # <- NUNCA hacer esto desde otro hilo
