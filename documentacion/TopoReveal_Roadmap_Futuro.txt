FASE 2 — Historial, Persistencia y Estadisticas
F2.1 — SQLite: Historial Diferencial entre Sesiones
Guardar un snapshot al final de cada sesion. Al iniciar la siguiente, comparar y mostrar que cambio: dispositivos nuevos, que desaparecieron, que cambiaron de tipo o puertos.
    • Tabla sessions: id, timestamp_inicio, interfaz, n_hosts, duracion
    • Tabla hosts: session_id, ip, mac, tipo, puertos_json, risk_score
    • Funcion diff_sesion(actual, anterior) → lista de cambios
    • Badge [NUEVO] [CAMBIO] [PERDIDO] visible en panel derecho
    • Export CSV con columna delta para auditorias
F2.2 — Dashboard de Estadisticas con Graficas
Ventana flotante con visualizaciones en tiempo real usando matplotlib embebido en tkinter (FigureCanvasTkAgg). Actualizacion cada 10s via after().
    • Pie chart: distribucion por tipo de dispositivo
    • Bar chart: distribucion de risk score por severidad
    • Line chart: hosts activos vs lobby por hora
    • Bar chart horizontal: top-10 puertos detectados en la red
    • Boton en barra superior para abrir/cerrar el dashboard

FASE 3 — Inteligencia Contextual
F3.1 — GeoIP de Conexiones Externas
Para cada IP externa detectada resolver pais + empresa ISP sin APIs remotas.
    • Libreria: geoip2 + base de datos GeoLite2 (descarga libre de MaxMind)
    • Mostrar bandera + empresa en panel de conexiones externas
    • Alerta especial si hay conexiones a paises inusuales para el contexto
    • Cache local por sesion para no repetir queries
F3.2 — Multi-Subnet / VLAN Detection
Detectar y visualizar subredes secundarias como hotspots WiFi o VLANs.
    • Detectar hosts con IP fuera de la subred primaria (ej: 192.168.43.x)
    • Dibujar como "isla" separada en el canvas con linea punteada al gateway
    • Etiquetar automaticamente como VLAN / HOTSPOT / EXTERNA segun patron
    • Contador de hosts por subred en estadisticas
F3.3 — Scanner Multi-Fase Adaptativo (Fase 0-3)
Sistema de escaneo en 4 fases progresivas segun lo que se descubre, minimizando ruido en la red.
    • Fase 0: Solo captura pasiva, sin nmap — modo silencioso
    • Fase 1: ARP sweep basico — descubrir hosts activos
    • Fase 2: nmap top-20 puertos por host nuevo detectado
    • Fase 3: Vuln scan + service versions para hosts con risk_score > 50
    • Transicion automatica segun umbrales de tiempo y risk

FASE 4 — Arsenal Avanzado y Reporting
F4.1 — Informe PDF Profesional
Generar informe ejecutivo en PDF al finalizar cada sesion.
    • Portada: timestamp, red escaneada, total hosts, duracion sesion
    • Tabla resumen de riesgos ordenada por severidad descendente
    • Captura PNG del canvas incluida en el informe
    • Recomendaciones automaticas por tipo de hallazgo (desde ACCIONES dict)
    • Libreria: reportlab o weasyprint
F4.2 — Notificaciones en Tiempo Real
Alertas visuales y opcionales para eventos criticos durante una sesion larga.
    • Toast notification cuando aparece host nuevo con riesgo alto o critico
    • Sonido configurable (beep corto) para alerta critica
    • Webhook opcional: enviar alerta JSON a Slack / Discord / Telegram
    • Toggle "modo silencioso" en la barra de herramientas
F4.3 — CVE Lookup Offline
Para cada servicio detectado (SSH, SMB, Telnet...) buscar CVEs conocidas.
    • Base de datos NVD descargada localmente (JSON feeds de NIST)
    • Correlacion: servicio + version → CVEs aplicables
    • Mostrar CVE ID + CVSS score + descripcion en el tooltip del hallazgo
    • Actualizar base NVD con un boton "Actualizar CVEs"

LLUVIA DE IDEAS — Features Adicionales
Ideas Grandes (alto impacto)
    • Modo Stealth: Solo captura pasiva, cero ARP activo. Para auditorias donde no se quiere generar trafico. Toggle en UI.
    • Timeline Replay: Reproducir la sesion como video. Slider de tiempo → el canvas muestra que habia en cada momento.
    • Deteccion de Gemelos: Dos MACs con la misma IP = posible IP spoofing → alerta critica automatica.
    • Dark Hunter Mode: Buscar activamente dispositivos que no responden ARP pero si TCP. nmap -sS a rangos silenciosos.
    • Servidor Web embebido: Exponer el canvas como pagina HTML via Flask para monitoreo remoto desde otro dispositivo.
    • Correlacion de Sesiones: Mismo MAC en diferentes subredes entre sesiones = dispositivo movil → rastrear sus IPs historicas.
    • Fingerprinting Avanzado: Integrar p0f pasivo para identificar OS con precision sin nmap activo.
Ideas Rapidas (quick wins, menos de 1 sesion)
    • Badge "NUEVO" verde parpadeante en el canvas los primeros 30s de un host
    • Boton "Copiar IP" en el menu click derecho del canvas
    • Tooltip: mostrar hace cuantos minutos fue visto por ultima vez
    • Color del enlace segun protocolo: ARP=amarillo, TCP=verde, UDP=azul
    • Doble-click en nodo → abrir Arsenal con Quick Ports preseleccionado
    • Filtro por rango de IP en panel derecho (ej: solo .200-.250)
    • Modo compacto automatico: reducir iconos a 12px cuando hay mas de 150 hosts
    • Contador de ARP scanners activos en la barra de estado superior
    • Boton "Silenciar nodo" en click derecho — ignorar ese host en alertas futuras
    • "Sticky notes" en nodos: el analista escribe observaciones manuales sobre un host
